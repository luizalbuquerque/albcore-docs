# CTEC Rastreamento

Documentação completa da API de rastreamento veicular para clientes **CTEC**.
Integre rastreamento GPS em tempo real, alarmes, vídeo ao vivo e controle remoto da sua frota.

<div className="base-url-box">
  <span className="label">Base URL</span>
  <code>http://39.100.229.38:8088</code>
</div>

<Callout type="info">
Todas as requisições (exceto login) exigem o parâmetro **`jsession`**, obtido via endpoint de [Autenticação](/ctec/autenticacao).
</Callout>

---

## Início Rápido

<Steps>

### Autentique-se

Faça login para obter o token `jsession`:

```bash
curl "http://39.100.229.38:8088/StandardApiAction_login.action?account=SEU_USER&password=SUA_SENHA"
```

```json
{ "result": 0, "jsession": "cf6b70a3-c82b-4392-8ab6-bbddce336222" }
```

### Consulte seus veículos

Use o `jsession` para listar veículos:

```bash
curl "http://39.100.229.38:8088/StandardApiAction_queryUserVehicle.action?jsession=SEU_JSESSION"
```

### Rastreie em tempo real

Obtenha a posição GPS atual de um dispositivo:

```bash
curl "http://39.100.229.38:8088/StandardApiAction_getDeviceStatus.action?jsession=SEU_JSESSION&devIdno=DEVICE_ID&toMap=1"
```

</Steps>

## Formato de Resposta

Todas as respostas são **JSON** com encoding **UTF-8**.

| Campo | Tipo | Descrição |
|-------|------|-----------|
| `result` | `number` | `0` = sucesso. Outro valor = erro (ver [Códigos de Erro](/ctec/codigos-erro)) |
| `callback` | `string` | Presente quando `callback` é enviado na requisição (JSONP) |

## Parâmetros Comuns

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|:-----------:|-----------|
| `jsession` | `string` | Sim | Session ID retornado pelo login |
| `callback` | `string` | Não | Nome da função JSONP para chamadas cross-domain |

<Callout type="warning">
**Encoding de parâmetros:** valores com caracteres especiais (acentos, etc.) devem ser codificados com `encodeURI(encodeURI(valor))` via URL ou `encodeURI(valor)` via Ajax.
</Callout>
