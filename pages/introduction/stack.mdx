## Decisões Técnicas

### Por que PostgreSQL e não MySQL?
- JSON nativo (jsonb) — útil para configs por tenant
- Window functions para relatórios
- Melhor suporte a row-level security (futuro)

### Por que RabbitMQ e não Kafka?
- Menor overhead operacional
- Suficiente para volume inicial
- Migração para Kafka quando >1M msg/dia

### Por que JWT stateless?
- Horizontal scaling sem sessão compartilhada
- Sem round-trip ao banco para validar token
- Redis usado apenas para blacklist (logout) quando necessário
