## Camadas do Projeto

```
Presentation (Controllers, DTOs de request/response)
    в†“ depende de

    в†“ depende de
```

  Domain NEVER importa de Application.
</Callout>

### 1. Domain Layer (Pura)
Define as regras de negГіcio sem dependГЄncias:

```java
    private String tenantId;
```

### 2. Application Layer (Use Cases)
Orquestra domain + infrastructure:

```java
@Service
@Transactional
public class CrmService {
    private final IContactRepository repository;

    public ContactResponse createContact(CreateContactRequest request) {
        ContactEntity entity = new ContactEntity();
        entity.setTenantId(TenantContext.getTenantId());
        entity.setName(request.getName());
        entity.validate();

        ContactEntity saved = repository.save(entity);
        return mapper.toResponse(saved);
    }
}
```

### 3. Infrastructure Layer (PersistГЄncia)
Implementa a port:

```java

    private final ContactSpringDataRepository springRepository;

    @Override
    public ContactEntity save(ContactEntity entity) {
        ContactJpa jpa = toJpa(entity);
        ContactJpa saved = springRepository.save(jpa);
        return toDomain(saved);

```

### 4. Presentation Layer (API)
ExpГµe via REST:

```java
@RestController

    @PostMapping
```
