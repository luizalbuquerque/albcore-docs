## 2. Os 8 Roles вҖ” Sem prefixo, contexto define o domГӯnio

```java
public enum Role {
    OWNER,        // Dono (master=root, tenant=dono)
    MANAGER,      // Gerente (master=global, tenant=local)
    FINANCIAL,    // Financeiro
    COMMERCIAL,   // Comercial (contexto master)
    SELLER,       // Vendas (contexto tenant)
    SUPPORT,      // Suporte
    TEAM,         // Colaborador (contexto tenant)
    CLIENT        // Cliente final (contexto tenant)
}
```

<Callout type="info">
  **Mesma role, significado diferente pelo contexto:**
  - `OWNER + tenant_id=NULL` вҶ’ super-admin root da AlbCore
  - `OWNER + tenant_id="tenant_X"` вҶ’ dono do tenant X
</Callout>

## 3. RBAC + Permissions

**Permission = "modulo:recurso:acao"**

Exemplos:
- `"crm:contacts:read"`
- `"crm:contacts:write"`
- `"scheduling:appointments:*"`
- `"payment:reports:read"`
- `"*:*:*"` (OWNER вҖ” acesso total)

### PermissГөes PadrГЈo por Role

**MASTER (tenant_id = NULL, schema public):**
| Role | Permissions |
|------|------------|
| OWNER | `*:*:*` |
| MANAGER | `tenants:*:*`, `templates:*:*`, `modules:*:*` |
| FINANCIAL | `billing:*:*`, `reports:financial:read` |
| COMMERCIAL | `tenants:*:read`, `metrics:*:read`, `tenants:trial:write` |
| SUPPORT | `tenants:*:read` (read-only em qualquer tenant) |

**TENANT (tenant_id = "tenant_X", schema do tenant):**
| Role | Permissions |
|------|------------|
| OWNER | `*:*:*` |
| MANAGER | `crm:*:*`, `scheduling:*:*`, `users:*:read` |
| FINANCIAL | `payment:*:*`, `reports:financial:read` |
| SELLER | `crm:contacts:*`, `crm:leads:*` |
| SUPPORT | `crm:contacts:read`, `scheduling:appointments:read` |
| TEAM | `scheduling:appointments:read` |
| CLIENT | `scheduling:own:read`, `documents:own:read` |
